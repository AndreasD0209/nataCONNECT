# NataConnect

Personal finance that runs on hardware you own, not on someone else's server.

Built for the AMD Developer Hackathon: ACT II, Track 3 (Unicorn / Open Innovation).

## What This Is

NataConnect is a local first personal finance application built around four areas the user moves between in a single interface.

**Shield** protects payment cards with rules the user writes in plain language, such as asking before any purchase over a set amount, blocking unverified sellers, or restricting online shopping during certain hours. Every blocked transaction shows the exact reason it was stopped.

**Practice** is a trading simulator where users place buy and sell orders with simulated money and receive feedback on each decision, giving people a safe place to learn how markets behave before risking real capital.

**Guide** is a financial assistant that answers questions about saving and investing in plain language. It always makes clear it offers general information rather than personalised advice, and it never carries out a transaction on the user's behalf.

**Goals** turns savings targets and monthly spending limits into a small set of clear, adjustable numbers, showing how changing the monthly savings amount shifts every deadline in real time.

The long term direction is to run the whole application on a small computer the user owns, such as a Raspberry Pi, so financial data never has to leave hardware the user controls.

## AMD / Fireworks Compute Usage

The **Guide** feature is the part of NataConnect that uses AMD compute. Guide calls the **Fireworks AI API**, which serves large language models on **AMD Instinct GPUs**. Every answer Guide produces is generated by a model running on AMD hardware, not by a call to a third party proprietary API running on other infrastructure.

The integration lives in a single file:

```
src/lib/fireworksClient.ts
```

This file builds the request to `https://api.fireworks.ai/inference/v1/chat/completions`, sends the user's question along with a system prompt that enforces two rules (restate the question, then answer as general information only), and returns the parsed response to the Guide UI component at `src/components/GuidePillar.tsx`.

`FIREWORKS_API_KEY` is read from an environment variable at runtime and is never committed to the repository. See [Environment Variables](#environment-variables) below for how to configure it.

## Setup

### Prerequisites

- Node.js 18 or later
- npm
- A Fireworks AI account and API key ([fireworks.ai](https://fireworks.ai))
- A Finnhub API key ([finnhub.io](https://finnhub.io)) for live market data in Practice

### Clone

```bash
git clone https://github.com/entercrystal/nataCONNECT.git
cd nataCONNECT
```

### Environment Variables

Inside the `project/` folder, create a `.env.local` file:

```
VITE_FIREWORKS_API_KEY=your_fireworks_api_key
VITE_FINNHUB_KEY=your_finnhub_api_key
```

Do not commit this file. It is already listed in `.gitignore`.

### Run Locally (frontend only, recommended for judging)

```bash
cd project
npm install
npm run dev
```

The app starts at `http://localhost:5173` by default. Open it in a modern browser.

### Build for Production

```bash
cd project
npm run build
```

Output is written to `project/dist/`.

### Run on a Raspberry Pi / Self-Hosted

```bash
npm install
node server.js
```

This starts the optional local server used for the self-hosted, local-first deployment path described above.

## External Services

- **Fireworks AI** (`api.fireworks.ai`) — LLM inference for the Guide feature, served on AMD Instinct GPUs. Requires `VITE_FIREWORKS_API_KEY`.
- **Finnhub** (`finnhub.io`) — live market data (current, highest, lowest price, percentage change) used in Practice. Requires `VITE_FINNHUB_KEY`.

No other external service, database, or third party API is used beyond what is listed above. The application does not collect, transmit, or store personal data on any server.
